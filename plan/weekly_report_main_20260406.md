# 주간보고 관리 — 메인화면 설계

## 개요

GitLab 리포지토리의 커밋 로그와 변경 파일을 분석하여, 주 단위로 "어떤 기능이 수정/추가되었는지"를 요약하고 엑셀 보고서로 출력하는 주간보고 관리 시스템의 메인화면 설계 문서.

---

## 주요 컴포넌트 상세

### 1. 헤더 (Header)

| 요소 | 설명 |
|------|------|
| 로고 / 타이틀 | "주간보고 관리" |
| 기간 선택기 | 주 단위 DateRangePicker (월~일 고정), 이전주/이번주/다음주 버튼 |
| 사용자 정보 | 로그인한 사용자명, 로그아웃 버튼 |

---

### 2. 사이드바 (Sidebar) — 프로젝트 목록

| 요소 | 설명 |
|------|------|
| `+ 프로젝트 추가` 버튼 | 클릭 시 GitLab 프로젝트 등록 모달 오픈 |
| 프로젝트 아이템 | 프로젝트명, GitLab 아이콘, 클릭 시 해당 프로젝트 상세 뷰로 전환 |
| 활성 표시 | 선택된 프로젝트 하이라이트 |
| 삭제 버튼 | 호버 시 삭제(X) 아이콘 표시 |

**GitLab 프로젝트 등록 모달**
```
┌────────────────────────────────────────┐
│  GitLab 프로젝트 추가                  │
│                                        │
│  GitLab URL                            │
│  [https://gitlab.com/org/repo    ]     │
│                                        │
│  인증 방식                              │
│  ○ Personal Access Token               │
│  ● OAuth (GitLab 로그인)               │
│                                        │
│  ┌──────────────────────────────────┐  │
│  │ GitLab 계정으로 로그인하여 권한  │  │
│  │ 을 부여합니다.                   │  │
│  │ [GitLab으로 로그인]              │  │
│  └──────────────────────────────────┘  │
│                                        │
│  [취소]                                │
└────────────────────────────────────────┘
```

- URL 입력 후 GitLab API로 프로젝트 존재 여부 검증
- PAT 방식: 토큰 입력 필드 추가 표시
- OAuth 방식 (구현 완료):
  1. `GET /api/oauth/gitlab/authorize?gitlabUrl=` → GitLab OAuth URL 반환 (state 생성)
  2. 팝업 오픈 → 사용자 GitLab에서 권한 승인
  3. `GET /api/oauth/gitlab/callback?code=&state=` → 토큰 교환 후 postMessage로 팝업 종료
  4. 프론트엔드 메시지 수신 → `POST /api/projects` 자동 호출

### 3. 메인 콘텐츠 영역

#### 3-1. 탭 구성

| 탭 | 설명 |
|----|------|
| 전체 프로젝트 | 등록된 모든 프로젝트의 활동을 통합하여 표시 |
| 개별 프로젝트 | 사이드바에서 선택한 프로젝트만 표시 |

#### 3-2. 활동 요약 카드

각 프로젝트(또는 전체 통합)에 대해 아래 정보를 카드 형태로 표시:

```
┌─────────────────────────────────────────────────────────┐
│  [프로젝트명]  2026-03-30 ~ 2026-04-05                  │
│  커밋 12건 · 기여자 3명                                  │
│─────────────────────────────────────────────────────────│
│  📝 AI 요약 (편집 가능 영역)                             │
│                                                         │
│  • 회원 인증 모듈 리팩토링 — JWT 갱신 로직 개선         │
│  • 상품 검색 API 성능 개선 — 인덱스 추가 및 쿼리 최적화 │
│  • 주문 상태 알림 기능 추가 — WebSocket 연동            │
│                                                 [✏ 편집] │
│─────────────────────────────────────────────────────────│
│  변경 파일 목록  ▼ 펼치기                               │
│  (숨김 상태, 펼치면 파일 경로 목록 표시)                 │
└─────────────────────────────────────────────────────────┘
```

#### 3-3. AI 요약 편집

- `[✏ 편집]` 클릭 시 textarea로 전환 (인라인 편집)
- 수정 후 `[저장]` / `[취소]` 버튼
- 저장된 내용은 DB에 유지 (재분석해도 수동 편집본 우선)

#### 3-4. 변경 파일 목록 (접기/펼치기)

펼쳤을 때 표시:
```
src/main/java/com/example/auth/JwtService.java       +45 -12
src/main/java/com/example/order/OrderService.java    +23 -5
frontend/src/components/SearchBar.tsx                +67 -3
...
```

### 4. 엑셀 내보내기

| 요소 | 설명 |
|------|------|
| `[📥 엑셀 내보내기]` 버튼 | 화면 우하단 고정 또는 콘텐츠 영역 하단 |
| 내보내기 범위 | 현재 선택된 기간 + 탭(전체/개별) 기준 |
| 파일명 | `주간보고_2026-03-30_2026-04-05.xlsx` |

**엑셀 시트 구조**

| 시트 | 내용 |
|------|------|
| 전체 요약 | 기간, 전체 커밋 수, 프로젝트별 요약 |
| [프로젝트명] | 커밋 목록, AI 요약, 변경 파일 |

| 상태 | 저장소 | 설명 |
|------|--------|------|
| 선택 기간 | Zustand | 헤더 기간 선택기 상태 |

| 편집된 요약 | React Query mutation | 사용자가 수정한 AI 요약 저장 |

## API 연동 흐름

```
1. 프로젝트 등록
   POST /api/projects  { gitlabUrl, accessToken, authType }
   → GitLab API 검증 후 DB 저장

1-1. OAuth 인증 흐름 (OAUTH 방식 선택 시)
   GET  /api/oauth/gitlab/authorize?gitlabUrl=  → { authUrl, state }
   → 팝업 오픈 → 사용자 승인
   GET  /api/oauth/gitlab/callback?code=&state= → HTML(postMessage + window.close)
   → 프론트엔드 token 수신 → POST /api/projects 자동 호출

2. 주간 분석 조회
   GET /api/reports?projectId=&startDate=&endDate=
   → GitLab Commits API 호출 → AI 요약 생성 → 결과 반환

3. 요약 수정 저장
   PUT /api/reports/{reportId}/summary  { summary }

4. 엑셀 내보내기
   GET /api/reports/export?projectId=&startDate=&endDate=
   → .xlsx 파일 다운로드
```

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | React 19 + TypeScript + Vite |
| 상태관리 | Zustand (UI), React Query (서버) |
| UI 컴포넌트 | 기존 프로젝트 컴포넌트 패턴 준수 |
| Backend | Java 25 + Spring Boot 4.x |
| GitLab 연동 | GitLab REST API v4 |
| AI 요약 | LLM API (커밋 메시지 + diff 분석) |
| 엑셀 생성 | Apache POI |
