You are a senior product designer specializing in modern SaaS dashboards (2025 trends).
Redesign the following Git analytics dashboard UI to feel more modern, minimal, and premium.

## Context

* Product: Git commit digest dashboard
* Users: Developers, team leads
* Key features:

  * Project list (sidebar)
  * Commit history grouped by contributor
  * AI summary panel
  * Date range filter
  * Export (Excel)
* Current problems:

  * Feels visually heavy
  * Lacks hierarchy and focus
  * Not “trendy” or polished
  * Weak spacing, typography, and color system

## Goals

* Make the UI feel like a modern SaaS (Linear, Vercel, Notion, Stripe-inspired)
* Improve readability and information hierarchy
* Increase perceived quality (clean, calm, premium)
* Keep it developer-friendly (not overly decorative)

## Design Directions (VERY IMPORTANT)

Apply these modern trends:

1. Soft minimalism (less borders, more spacing)
2. Subtle glassmorphism or layered surfaces
3. Neutral base + 1 accent color (purple is OK but refine it)
4. Clean typography (Inter / Pretendard style hierarchy)
5. Card-based layout with clear grouping
6. Better density control (comfortable but information-rich)
7. Micro-interactions (hover, selection, expand states)

## Specific Improvements Required

### 1. Layout

* Redesign overall layout with clearer sections:

  * Sidebar (lighter, cleaner)
  * Main content (focus area)
  * Right summary panel (less heavy, more elegant)
* Improve spacing system (8px grid or similar)

### 2. Sidebar

* Reduce visual weight (lighter background or subtle contrast)
* Improve active state (modern highlight style)
* Add icons if helpful

### 3. Commit List

* Improve grouping by contributor
* Use better visual hierarchy:

  * Contributor name (strong)
  * Commit message (primary)
  * Date (secondary)
* Reduce borders, use spacing or dividers instead
* Add hover + expand interaction ideas

### 4. AI Summary Panel

* Make it feel like a smart assistant card
* Improve typography and spacing
* Add subtle visual cues (icon, highlight, tone)
* Avoid dense text blocks

### 5. Top Controls (Date, Filters)

* Make them compact and modern
* Use pill-style inputs or segmented controls

### 6. Visual Style

* Suggest:

  * Color palette (with hex)
  * Typography scale
  * Spacing system
  * Component styles (buttons, cards, tags)

### 7. Output Format

Provide:

1. High-level design concept
2. Before vs After improvements
3. Detailed UI description (section by section)
4. Design system (colors, typography, spacing)
5. Optional: TailwindCSS-style tokens or CSS variables

Be very specific and practical so a developer can implement it directly.
