1.프로젝트 폴더를 추가해서 하위에 프로젝트를 추가할 수 있다
2.프로젝트 폴더 설정을 한다
- 구분: 개발사업/신규추진사업/기타
- 프로젝트명
- 담당자: 여러명 선택 가능함

현재 개발 중인 시스템에 '프로젝트 관리 및 폴더 구조화' 기능을 추가하려고 해. 이 내용을 바탕으로 @plan/ 아래에 md 파일을 작성해줘.

1. 주요 요구 사항:

폴더 기반 관리: 사용자는 '프로젝트 폴더'를 생성할 수 있고, 해당 폴더 하위에 여러 개의 세부 프로젝트를 추가할 수 있는 계층 구조를 가짐.

프로젝트 설정 데이터:

구분: 개발사업, 신규추진사업, 기타 (Select 타입)

프로젝트명: 텍스트 입력

담당자: 시스템 사용자 목록에서 여러 명을 선택(Multi-select)할 수 있어야 함.

2. plan.md에 포함될 내용:

Features: 구현해야 할 기능 목록 (폴더 CRUD, 프로젝트 설정 UI, 담당자 검색 및 추가 로직 등).

Database Schema: 폴더(Folders)와 프로젝트(Projects)의 관계, 그리고 담당자 다중 선택을 위한 매핑 테이블(Project_Members)을 포함한 DB 설계 안.

API Design: 폴더 생성, 프로젝트 등록, 담당자 목록 조회 등을 위한 REST API 엔드포인트 설계.

UI/UX Plan: 사용자가 폴더를 직관적으로 관리하고 담당자를 다중 선택할 때의 흐름(예: 태그 형태의 칩 UI).

===

폴더 설정 데이터: 폴더 생성 및 수정 시 다음 정보를 설정함.

구분: 개발사업 / 신규추진사업 / 기타 (선택형)
프로젝트명: 해당 폴더를 대표하는 명칭
담당자: 시스템 사용자 중 다중 선택(Multi-select) 가능
상태: 진행중 / 완료 (상태 값 관리)
