# 사이드바 폴더 목록 그룹핑(Grouping) UI 고도화 요청

## 1. 개요
현재 왼쪽 사이드바의 `FOLDERS` 리스트를 폴더 설정의 '구분(Type)' 값에 따라 그룹화하여 표시함으로써, 사업 성격별로 폴더를 빠르게 찾을 수 있도록 개선합니다.

## 2. 데이터 구조 및 그룹핑 로직
- **그룹핑 기준**: 폴더 설정 시 선택한 `구분` (개발사업 / 신규추진사업 / 기타)
- **정렬 순서**: 
    1. 개발사업 그룹
    2. 신규추진사업 그룹
    3. 기타 그룹
    - 각 그룹 내에서는 폴더명이 가나다순으로 정렬되거나 최신 생성순으로 정렬되도록 로직 구성.

## 3. UI/UX 상세 설계
### A. 그룹 헤더(Group Header) 추가
- 사이드바 내에서 각 그룹의 시작점에 소제목(예: [개발사업], [신규추진사업])을 표시합니다.
- 그룹 헤더는 일반 폴더와 구분되도록 약간 작은 폰트나 연한 색상, 혹은 좌측에 작은 구분선을 활용합니다.

### B. 계층 구조 유지
- **1단계**: 그룹 헤더 (예: 개발사업)
- **2단계**: 폴더명 (예: KT 사장이지 쇼핑센터 구축) + 진행상태 배지
- **3단계**: 세부 프로젝트명 (예: kt-sjeasy-shopping-center-api)

### C. 접기/펴기 기능 (Optional)
- 특정 그룹(예: 기타)의 목록이 길 경우, 그룹 헤더를 클릭하여 해당 그룹에 속한 폴더 리스트를 한꺼번에 접거나 펼칠 수 있는 아코디언 기능을 추가합니다.

## 4. 기술적 구현 요청
1. **데이터 가공 (React)**:
   - `folders` 배열을 받아 `lodash`의 `groupBy` 혹은 Native JavaScript의 `reduce`를 사용하여 `{ "개발사업": [...], "기타": [...] }` 형태의 객체로 변환하는 로직을 작성해줘.
2. **컴포넌트 리팩토링**:
   - `Sidebar` 컴포넌트 내에서 그룹화된 객체를 `Object.entries().map()`으로 순회하며 렌더링하도록 코드를 수정해줘.
3. **스타일링 (Tailwind CSS)**:
   - 그룹 간의 간격(Margin)을 적절히 주어 사업군별로 영역이 명확히 구분되게 해줘.

## 5. 제약 사항
- 이미지에 보이는 기존의 '진행중/완료' 배지 디자인과 트리 구조 스타일을 그대로 유지할 것.
- 폴더가 하나도 없는 그룹은 헤더를 숨기거나 '내역 없음'을 표시할 것.
