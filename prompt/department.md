# 조직 계층 구조(실-팀) 도입을 위한 DB 및 로직 설계 요청

현재 `teams` 테이블만 존재하는 구조에서, 그 상위 개념인 **'실(Department)'** 테이블을 추가하여 조직 체계를 확장하려고 합니다. 

## 1. DB 스키마 설계

### A. departments (실 테이블) 생성
`teams` 테이블의 상위 개념으로, 실 정보를 저장합니다.
- **id:** PK (BigInt)
- **name:** 실 이름 (예: 플랫폼개발실, 서비스개발실, 솔루션연구소 등)
- **created_at / updated_at:** 생성/수정일시

### B. teams (팀 테이블) 수정
어느 '실'에 소속되어 있는지 구분하기 위해 외래키(FK)를 추가합니다.
- **dept_id:** FK (departments.id 참조)

## 2. 계층 데이터 예시
| 실(Department) | 팀(Team) |
| :--- | :--- |
| **플랫폼개발실** | 플랫폼개발팀 |
| **서비스개발실** | 서비스개발1팀, 서비스개발2팀, 서비스개발3팀 |
| **솔루션연구소** | 솔루션개발1팀, 솔루션개발2팀 |
