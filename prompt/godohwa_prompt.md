# 프로젝트 페르소나 및 기능 고도화 요청

## 1. 프로젝트 개요
- **서비스명**: Hubilon GitDigest (사업별 프로젝트 통합 관리 시스템)
- **UI 목표**: 사업(폴더) 내 프로젝트들의 가시성 확보 및 보고서 자동화

## 2. 중앙 섹션 상세 설계: [View Mode 분리]
사용자가 상단 탭을 통해 '사업 전체'를 볼지, '특정 프로젝트'를 볼지 선택합니다.

### A. 전체 프로젝트 뷰 (Total View) - [사업 단위 스캔]
- **목적**: 해당 사업(폴더)에 속한 모든 프로젝트의 상태를 한 페이지에서 확인.
- **표시 방식**: 
    - 폴더에 속한 모든 프로젝트를 **개별 카드 형태**로 세로로 나열 (Vertical Stack).
    - 각 카드 상단에 프로젝트명, 커밋 수, 기여자 수를 요약 표시.
    - 각 프로젝트 카드의 하단에 해당 프로젝트의 최신 커밋 이력을 나열.
- **기능**: 여러 프로젝트의 카드를 한 번에 스캔하며 사업 전체의 흐름을 파악.

### B. 개별 프로젝트 뷰 (Individual View) - [단일 프로젝트 집중]
- **목적**: 특정 프로젝트 하나만 선택하여 상세 내역 확인.
- **표시 방식**: 
    - 사이드바나 리스트에서 선택된 **단 하나의 프로젝트** 카드만 화면에 크게 표시.
    - 해당 프로젝트의 전체 커밋 히스토리를 심층적으로 확인하는 용도.

## 3. 우측 섹션: [통합 AI 보고서]
- **데이터 통합**: 현재 중앙 섹션에 노출된(혹은 체크된) **모든 프로젝트의 커밋 메시지를 통합**하여 요약.
- **요약 대상**: 개별 프로젝트가 아닌 '사업 폴더' 전체를 하나의 단위로 보고서 작성.
- **타이틀**: 현재 선택된 '사업 폴더명'을 보고서 제목으로 사용.

## 4. 폴더 설정 데이터 (Folder Meta)
- **구분**: 개발사업 / 신규추진사업 / 기타
- **프로젝트명**: 사업 폴더 전체 명칭
- **담당자**: 다중 선택 가능 (칩 UI 적용)
- **상태**: 진행중 / 완료 (배지 표시)

## 5. 기술적 구현 요청
1. **React Component 구조**: 
    - `TotalView`: `Folder.projects.map()`을 사용하여 여러 `ProjectCard`를 렌더링.
    - `IndividualView`: `selectedProjectId`에 해당하는 단일 `ProjectCard`만 렌더링.
2. **AI 데이터 취합 로직**: 
    - `const allCommits = selectedFolder.projects.flatMap(p => p.commits)` 형태의 통합 로직 구현.
3. **UI 톤앤매너**: 기존의 깔끔한 화이트/블루 테마를 유지하며, 카드 간 간격(Margin)과 구분선을 명확히 하여 가독성 확보.
