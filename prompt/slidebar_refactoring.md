# UI/UX 개선 및 기능 추가 지시서

## 1. 개요
현재 개발 중인 'Hubilon GitDigest' 서비스의 UI를 개선하고, 특정 페이지에서 누락된 사이드바(Sidebar)를 통합하는 작업을 수행합니다. 

## 2. 참조 이미지 분석
- **"@/prompt/image/main_page.png"**: 메인 대시보드 화면. 좌측 사이드바와 프로젝트 리스트, 우측 주간보고 상세 영역으로 구성됨.
- **"@/prompt/image/weekly_scheduler_page.png"**: '주간보고 스케줄러' 화면. 현재 사이드바가 보이지 않고 전체 화면을 사용 중임.

## 3. 수정 및 추가 요청 사항

### ① 사이드바(Sidebar) 기능 개선
- **열기/접기(Toggle) 기능 추가**: 
    - 사이드바를 접고 펼 수 있는 토글 버튼을 추가해 주세요.
    - 접었을 때는 아이콘만 표시되거나 좁은 형태로 유지되어 콘텐츠 영역을 넓게 확보할 수 있어야 합니다.
- **가로 최소 길이(min-width) 설정**: 
    - 사이드바가 일정 크기 이하로 줄어들어 텍스트가 깨지거나 UI가 무너지지 않도록 적절한 `min-width` 값을 설정해 주세요.
- **배경색 변경**: 
    - 현재 연한 회색 톤인 사이드바 배경색을 **흰색(#FFFFFF)**으로 변경하여 전체적으로 깔끔한 느낌을 주세요.
    - 배경색 변경에 따라 필요 시 테두리(Border)나 그림자(Shadow)를 추가하여 콘텐츠 영역과 구분해 주세요.

### ② 주간보고 스케줄러 화면 개선
- **사이드바 노출**: 
    - 현재 `image_39ab4c.png` 화면에는 사이드바가 없습니다. 대시보드와 동일한 레이아웃을 적용하여 좌측에 사이드바가 항상 노출되도록 레이아웃 구조를 변경해 주세요.
    - 사이드바와 우측 스케줄러 테이블 콘텐츠가 1:N 비율로 적절히 배치되어야 합니다.

## 4. 기술적 요구 사항
- 반응형을 고려하여 사이드바가 접혔을 때 우측 콘텐츠 영역이 유연하게 늘어나도록 구현해 주세요.
- 사이드바 상태(열림/닫힘)는 로컬 상태 관리 또는 Context를 사용하여 페이지 이동 시에도 유지될 수 있으면 좋습니다.
