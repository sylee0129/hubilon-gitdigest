## 3. 우측 섹션: Gemini API 기반 통합 보고서 고도화

### A. 엑셀 양식 동기화 (Style Alignment)
- **참조**: `@sample/03월2주_주간보고_플랫폼개발실_20260313.xlsx`
- **핵심 요구사항**: 
    - AI는 단순히 커밋을 요약하는 것이 아니라, 위 엑셀 파일에 기재된 **'문장 수준'**과 **'항목 분류 방식(사업관리/개발/QA)'**을 그대로 복제해야 함.
    - 엑셀에 적힌 "KT보안성 검토 이슈 확인 및 수정"과 같이 구체적이고 전문적인 용어를 사용하도록 유도.

### B. Gemini API 프롬프트 수정
- 기존의 단순 포맷팅 방식에서 탈피하여, **PM 페르소나**를 명확히 부여함.
- **차주 계획 추론 로직**: 
    - 개발 커밋이 많으면 -> '통합 테스트 및 버그 수정 예정'
    - 기획/설계 커밋이 많으면 -> '개발 환경 구축 및 퍼블리싱 착수 예정'
    - 이와 같이 현재 상태를 분석하여 미래 작업을 제안하도록 구성.

### C. 데이터 바인딩
- AI 응답값은 향후 엑셀 라이브러리(Apache POI 등)를 통해 엑셀의 특정 셀에 바로 입력될 수 있도록 `String` 또는 `JSON` 형태로 정교하게 관리함.

#### D. `summarizeFolder()` 수정

```
당신은 전문적인 IT 프로젝트 매니저(PM)이자 주간보고서 작성 전문가입니다.
제공된 [커밋 목록]을 분석하여, 첨부된 엑셀 샘플(@sample)의 작성 스타일과 완벽히 일치하는 보고서를 작성하세요.

[작성 가이드라인]
1. 문체: 반드시 "~함", "~완료", "~예정"으로 끝나는 격식 있는 개조식 문체를 사용함 (예: '수정 완료', '계획 수립', '검토 진행').
2. 구조화: 사업(폴더) 내의 작업들을 '사업관리', '개발', 'QA/테스트' 등의 카테고리로 논리적으로 분류하여 정리함.
3. 상세도: 단순히 커밋 메시지를 나열하지 말고, 비즈니스 관점에서 어떤 기능이 완성되었는지 요약함.
4. 차주 계획: 금주 커밋의 연속성을 고려하여, 다음 단계(예: 연동 테스트, 운영 배포, 보안성 검토)를 PM의 관점에서 논리적으로 추론하여 작성함.
5. 금지 사항: 인사말, 설명, 머리말/꼬리말은 절대 포함하지 말고 아래 [출력 형식]만 출력함.

[출력 형식]
[금주 진행사항 (%s~%s)]
- %s (사업명)
    > 사업관리
    : 내용 입력
    > 개발
    : 내용 입력

[차주 진행계획 (%s~%s)]
- %s (사업명)
    > 내용 입력 (금주 작업과 이어지는 논리적 단계)

[커밋 목록]
%s
```
