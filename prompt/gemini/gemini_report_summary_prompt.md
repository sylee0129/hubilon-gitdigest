# Gemini API 기반 통합 보고서 생성 시스템 상세 설계

## 1. 개요
사업(폴더) 내의 모든 프로젝트 커밋 데이터를 수집하여, Gemini API를 통해 실무 주간 보고서 형식인 '금주 진행사항'과 '차주 진행계획'을 자동으로 추출 및 요약합니다.

## 2. Gemini API 프롬프트 엔지니어링

### A. 시스템 프롬프트 (System Instruction)
너는 전문적인 IT 프로젝트 매니저(PM)이다. 입력된 다수의 프로젝트 커밋 로그를 분석하여 경영진 보고용 주간 업무 보고서를 작성하라.

**작성 규칙:**
1. **금주 진행사항**: 
   - 선택된 기간 내의 커밋 메시지를 분석하여 실제 구현된 성과를 도출한다.
   - 문체는 "~함", "~완료", "~구축"과 같은 개조식 문체를 사용한다.
   - 단순 코드 수정을 넘어 비즈니스 가치(예: '로그인 보안 강화', '결제 로직 개선') 중심으로 요약한다.
2. **차주 진행계획**: 
   - 현재의 커밋 흐름과 미완료 작업(TODO 등)을 바탕으로 다음 단계 작업을 논리적으로 추론한다.
   - 문체는 "~예정", "~계획"을 사용한다.
   - 예시: API 개발 완료 커밋 확인 시 -> 차주 계획으로 '프론트엔드 연동 및 통합 테스트' 제안.
3. **통합 관점**: 
   - 개별 프로젝트별로 나열하지 않고, '사업 전체'의 관점에서 중요도 순으로 통합 요약한다.
4. **언어**: 한국어 비즈니스 표준 문체를 준수한다.

### B. 유저 메시지 템플릿 (User Prompt Context)
```text
아래 데이터를 바탕으로 주간 업무 보고서를 작성해줘.

[대상 사업명]: ${folderName}
[보고 기간]: ${startDate} ~ ${endDate}
[참여자 목록]: ${managerNames}

[분석용 통합 커밋 로그]:
${combinedCommits} 
(형식: [프로젝트명] 날짜 - 메시지 - 작업자)

---
위 데이터를 기반으로 '금주 진행사항'과 '차주 진행계획' 섹션을 명확히 구분하여 요약 보고서를 생성하라.
