# AI 보고서 요약 기능 리팩토링 가이드 (v1.1)

## 1. 개요
현재 프로젝트별로 수행되던 AI 요약 기능을 **사업(폴더) 단위 통합 요약**으로 고도화합니다. Gemini API를 사용하여 실무 보고서 수준의 퀄리티를 확보하는 것이 핵심입니다.

## 2. 참조 문서 (Context)
클로드는 아래 두 파일을 반드시 결합하여 분석해야 합니다.
1. **프롬프트 가이드**: `@plan/gemini/gemini_report_summary_prompt.md`
   - Gemini API 호출을 위한 시스템 페르소나 및 상세 규칙 정의.
2. **출력 양식 가이드**: `@sample/03월2주_주간보고_플랫폼개발실_20260313.xlsx`
   - 실제 보고서에서 사용되는 문체(예: "~함", "~완료"), 섹션 구성, 내용의 상세 수준(Granularity)을 정의하는 기준점.

## 3. 리팩토링 상세 지시

### A. AI 출력 톤앤매너 (Style Matching)
- **엑셀 데이터 분석**: 엑셀 내 '진행사항'과 '진행계획'에 적힌 텍스트의 스타일을 학습하여, AI가 생성하는 텍스트가 위질감 없이 엑셀에 바로 붙여넣기 될 수 있도록 함.
- **섹션 대응**: 
    - 엑셀의 '진행사항 (MM/DD ~ MM/DD)' -> Gemini의 '금주 진행사항'
    - 엑셀의 '진행계획 (MM/DD ~ MM/DD)' -> Gemini의 '차주 진행계획'

### B. 백엔드 및 프롬프트 로직
- **커밋 데이터 전처리**: 폴더 내 여러 프로젝트의 커밋을 엑셀의 '구분'별(개발사업/신규추진 등)로 묶어서 요약할 수 있도록 데이터 구조화.
- **보고서 자동 채우기**: AI가 요약한 결과가 향후 '엑셀 내보내기' 기능을 통해 해당 엑셀 양식의 정확한 셀(Cell) 위치에 매핑될 수 있도록 필드값을 설계.

### C. UI/UX 개선
- **보고서 뷰어**: 우측 섹션에 엑셀 양식과 유사한 형태의 프리뷰 제공.
- **편집기**: 사용자가 AI 요약본을 수정한 내용이 최종 엑셀 파일에 반영되도록 상태 관리 로직 구현.

## 4. 최종 결과물 기대치
- 사용자가 폴더를 선택하고 [AI 요약] 클릭 시, **샘플 엑셀의 스타일을 그대로 재현한** 주간 보고서 초안이 생성되어야 함.
- 생성된 보고서는 '진행사항'과 '진행계획'이 명확히 구분되어야 하며, 사업 전체의 핵심 성과를 포함해야 함.
