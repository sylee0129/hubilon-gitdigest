# [Task] 주간보고서 엑셀 다운로드 기능 구현 및 레이아웃 설계

## 1. 목적
Hubilon GitDigest의 프로젝트 활동 데이터를 기반으로, 경영진 및 실장님 보고용 전문 엑셀 레이아웃을 설계하고 다운로드 기능을 구현한다.

## 2. 데이터 소스 (Input)
* **사업 구분:** [개발사업, 신규추진사업, 기타] 등 최상위 카테고리
* **프로젝트명:** 각 카테고리 내 프로젝트 명칭
* **담당자:** 프로젝트별 참여 인원 리스트 
* **주차 정보:** 현재 선택된 주차 기간 (예: 2026-04-06 ~ 2026-04-12)
* **내용:** [금주 진행사항], [차주 진행계획] (데이터 없을 시 '진행사항 없음', '진행사항 확인'으로 대체)

## 3. 클로드에게 위임하는 레이아웃 설계 (Requirements)
클로드는 아래 조건에 부합하는 **최적의 엑셀 레이아웃**을 직접 설계하여 `exceljs` 라이브러리로 구현하십시오.

### [디자인 및 레이아웃 가이드라인]
1. **전문성:** 사내 공식 보고서로 사용하기 적합한 깔끔하고 가독성 높은 스타일링.
2. **시트 구성:** 시트명에 주차정보(예: 03월 02주)를 포함할 것.
3. **헤더 설계:** - 1행 또는 2행을 활용하여 '주간 프로젝트 진행 현황'과 같은 타이틀 섹션 구성.
   - 각 컬럼 헤더에 해당 주차의 날짜 범위를 동적으로 삽입.
4. **가독성 최적화:** - 텍스트가 긴 '진행사항' 컬럼은 줄바꿈(Wrap Text) 및 너비 확장 적용.
   - 행 높이를 적절히 조절하여 내용이 잘리지 않도록 함.
5. **시각적 요소:** - 헤더 배경색(배색) 및 경계선(Border) 설정.
   - 사업 구분별로 셀 병합(Merge) 또는 구분선 처리를 통해 가독성 극대화.
6. **엑셀 파일명:** - 주차정보_주간보고_플랫폼개발실_날짜.xlsx (예: 03월2주_주간보고_플랫폼개발실_20260313.xlsx)

## 4. 기술 스택
* **Language:** TypeScript / JavaScript
* **Library:** `exceljs` (스타일링 자유도가 높으므로 우선순위 권장)
* **Output:** Blob 형태의 엑셀 파일 다운로드

## 5. 최종 결과물
* 사용자가 '주간보고 다운로드' 버튼을 클릭했을 때, 클로드가 설계한 레이아웃이 적용된 파일이 즉시 다운로드되어야 함.
