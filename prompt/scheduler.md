# 주간보고 자동 업로드 스케줄러 구현 요청

현재 '보고서 생성' 및 'Confluence 업로드' 기능은 구현이 완료된 상태입니다. 
이 기능들을 조합하여 **매주 목요일 19시**에 자동으로 동작하는 **스케줄러 모듈** 작성을 요청합니다.

## 1. 스케줄러 설정
- **실행 시점:** 매주 목요일 19:00 (KST)
- **대상 범위:** 시스템 내 '진행 중' 상태인 모든 개발 사업 폴더

## 2. 핵심 동작 프로세스 (분기 로직)
스케줄러가 실행되면 각 사업 폴더별로 아래 순서에 따라 데이터 존재 여부를 체크하고 로직을 수행해야 합니다.

### [Step 1] 폴더 보고서 데이터 확인
- DB에서 해당 사업의 '금주 진행사항' 및 '차주 진행계획' 입력 값을 확인합니다.

### [Step 2] 조건별 실행 (핵심 요구사항)
1. **내용이 있는 경우 (Manual 작성 건):**
   - 사용자가 대시보드에서 직접 내용을 입력하고 저장한 상태입니다.
   - **동작:** AI 요약을 수행하지 않고, 저장된 텍스트를 그대로 사용하여 confluence 업로드 기능을 수행합니다.
   
2. **내용이 없는 경우 (Empty 상태):**
   - 사용자가 보고서를 작성하지 않은 상태입니다.
   - **동작:** 하위 프로젝트들의 Git 커밋 이력을 수집하여 **AI 요약 기능(기구현)**을 먼저 실행한 후, 생성된 결과물로 confluence 업로드 기능을 수행합니다.

## 3. 데이터 구조 참고 (Hubilon GitDigest 기반)
- **사업명:** 예) KT 사장이지 쇼핑센터 구축
- **하위 프로젝트:** `kt-sjeasy-shopping-center-api`, `kt-shopping-center` 등
- **수집 데이터:** 커밋 수, 기여자 정보(Jisoo, youngkwanji 등), 커밋 메시지

## 4. 요청 사항
- 이미 구현된 AI 요약, 주간보고 다운로드, Confluence 업로드 기능을 배치(Batch) 형태로 실행하는 **스케줄러 루프 로직**을 작성해줘.
- 에러 발생 시 해당 사업 폴더만 건너뛰고 다음 사업을 계속 진행하는 예외 처리를 포함해줘.
- 발생한 에러는 대시보드 화면에서 확인 가능하도록 처리해줘.
