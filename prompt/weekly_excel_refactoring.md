# 주간 보고 엑셀 내보내기 리팩토링 요청

## 1. 개요
현재 '진행 중'인 사업 데이터를 추출하여 주간업무보고 엑셀 파일을 자동 생성하는 기능을 리팩토링하고자 합니다. 

## 2. 엑셀 레이아웃 기준
- **참조 파일**: `@sample/03월2주_주간보고_플랫폼개발실_20260313.xlsx`
- **구현 원칙**: 위 샘플 파일의 모든 시각적 요소(폰트, 배경색, 테두리, 열 너비, 행 높이)를 그대로 재현해야 합니다.
    - **타이틀**: 1행에 "부서명 + 날짜 + 주간업무보고" 형식 유지.
    - **헤더**: 3행의 '구분', '진행사항', '진행계획', '담당자' 양식 적용.
    - **셀 병합**: 동일한 '구분' 값을 가진 프로젝트들은 샘플처럼 세로로 셀을 병합 처리.
    - **텍스트 스타일**: 셀 내 자동 줄바꿈(Wrap Text) 및 상단 정렬 적용.

## 3. 데이터 추출 소스 및 매핑 규칙
데이터는 크게 '폴더 내 메타 정보'와 '보고서 파일 텍스트' 두 곳에서 가져와 매핑합니다.

| 엑셀 항목 | 데이터 추출 소스 (Source) | 비고 |
| :--- | :--- | :--- |
| **구분** | **폴더 데이터** (Project Metadata) | 사업 분류 (예: 개발 사업, 신규 추진 사업, 기타 등) |
| **프로젝트명** | **폴더 데이터** (Project Metadata) | 각 행의 제목으로 사용 |
| **진행사항** | **폴더 보고서** (Report Content) | 보고서 내 '진행사항' 섹션 텍스트 단순 추출 |
| **진행계획** | **폴더 보고서** (Report Content) | 보고서 내 '진행계획' 섹션 텍스트 단순 추출 |
| **담당자** | **폴더 데이터** (Project Metadata) | 프로젝트 참여 인원 (샘플처럼 줄바꿈으로 구분) |

## 4. 기술 요구사항
- **기술 스택**: Java/Spring Boot (Apache POI 라이브러리 활용) 기반.
- **주요 로직**:
    1. **필터링**: 상태값이 '진행 중'인 사업만 추출 대상으로 함.
    2. **텍스트 파싱**: 각 프로젝트 폴더에 저장된 보고서 파일에서 '진행사항'과 '진행계획' 텍스트를 읽어오는 로직 구현.
    3. **동적 날짜**: 실행 시점의 날짜를 기준으로 헤더에 주간 날짜 범위(예: 03/09 ~ 03/13)를 자동 계산하여 표시.
- **코드 품질**: 
    - 스타일 설정(CellStyle) 로직과 데이터 쓰기 로직을 분리하여 재사용성 확보.
    - 대량 데이터 처리를 고려한 메모리 관리 로직 포함.

## 5. 최종 결과물 요청
- 위 샘플 레이아웃을 완벽하게 재현하는 **Excel Export Service** 코드.
- 폴더 기반 데이터와 보고서 텍스트를 조합하는 **DTO 모델 및 데이터 매핑 로직**.
- 유지보수가 용이하도록 스타일과 데이터가 분리된 구조 설계안.
