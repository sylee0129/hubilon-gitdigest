# 로그인 계정 팀 기반 폴더 세팅(Filtering) 리팩토링 요청

사용자가 로그인 후 메인 페이지에 진입했을 때, 해당 사용자가 속한 팀(Team)의 폴더들만 화면에 표시되도록 시스템을 개선하고자 합니다.

## 1. 관련 데이터 구조 (DB)
- **teams 테이블:** 팀 정보 관리 (`id`, `name` 등)
  - 예: `id: 1` / `name: 플랫폼개발팀`
- **folders 테이블:** 각 사업 폴더 정보 (`id`, `name`, `status`, `category_id` 등)
  - 현재 이미지상으로는 `team_id` 외래키가 보이지 않으나, 논리적으로 **folders 테이블에 `team_id` 컬럼이 추가되어 있거나 중간 매핑 테이블이 존재함**을 가정합니다.

## 2. 핵심 요구사항
- **팀별 격리:** 로그인한 사용자가 '플랫폼개발팀' 소속이라면, 사이드바의 `FOLDERS` 영역에는 해당 팀과 연관된 폴더(예: KT 사장이지 쇼핑센터 구축, 프로젝트A 등)만 로드되어야 합니다.
- **초기 세팅:** 페이지 진입 시점에 사용자의 `team_id`를 기반으로 API를 호출하여 폴더 리스트를 셋팅합니다.

## 3. 구현 프로세스 가이드
1. **사용자 정보 추출:** 로그인 시 반환된 JWT 또는 세션 정보에서 사용자의 `teamId`를 가져옵니다.
2. **API 요청 수정:** - 기존: `GET /api/folders` (모든 폴더 조회)
   - 변경: `GET /api/folders?teamId={loginUserTeamId}` (특정 팀 폴더만 조회)
3. **Frontend(React) 반영:**
   - 메인 대시보드 컴포넌트의 `useEffect` 내에서 사용자의 팀 정보를 기반으로 폴더 데이터를 상태(State)에 저장합니다.
   - 사이드바의 카테고리별(개발 사업, 신규 추진 사업 등) 폴더 개수 카운트도 해당 팀의 데이터로 업데이트합니다.
