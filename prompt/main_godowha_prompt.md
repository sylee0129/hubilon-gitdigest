# 프로젝트 페르소나 및 메인 진입 화면 고도화 요청

## 1. 프로젝트 개요
- **서비스명**: Hubilon GitDigest
- **수정 목표**: 로그인 후 초기 진입 시 빈 화면 대신 '전체 사업 요약' 대시보드를 제공하고, 선택 시에만 상세 데이터를 노출하여 사용자 경험(UX) 개선.

## 2. 화면 상태 분기 (State Management)
중앙 섹션은 `selectedFolder`와 `selectedProject`의 유무에 따라 두 가지 모드로 동작합니다.

### A. [초기 진입 모드] 사업 전체 현황 대시보드
- **트리거**: 선택된 폴더나 프로젝트가 없는 경우 (null/undefined).
- **표시 내용**:
    1. **사업 요약 통계**: 
       - 총 관리 사업(폴더) 수, 현재 '진행중'인 사업 수 표시.
       - 금일 혹은 이번 주 전체 프로젝트에서 발생한 총 커밋 수 합산 데이터.
    2. **최근 업데이트 사업**: 
       - 최근 24시간 내에 커밋이 발생한 폴더 리스트 TOP 3 표시.
    3. **가이드 섹션**: 
       - "분석할 사업 폴더나 개별 프로젝트를 왼쪽 사이드바에서 선택하여 상세 이력과 AI 보고서를 확인하세요." 라는 시각적 가이드 메시지.

### B. [상세 내역 모드] 커밋 이력 및 보고서
- **트리거**: 사이드바에서 폴더(사업) 혹은 개별 프로젝트를 클릭한 경우.
- **표시 방식**: 
    - **폴더 선택 시**: 해당 사업 내의 모든 프로젝트 카드들이 세로로 나열 (사업 단위 통합 요약 가능).
    - **프로젝트 선택 시**: 해당 단일 프로젝트의 상세 커밋 히스토리 집중 표시.

## 3. 우측 섹션: [보고서 영역] 가변 처리
- **상태**: 초기 진입 모드에서는 보고서 영역을 비워두거나, "사업을 선택하면 AI 요약 보고서가 여기에 생성됩니다."라는 안내 텍스트 노출.
- **활성화**: 폴더/프로젝트 선택 후 [AI 요약] 버튼을 눌렀을 때만 실제 보고서 내용 렌더링.

## 4. 기술적 구현 요청
1. **Conditional Rendering**:
   - `if (!selectedId) { return <DashboardView />; }` 형태의 조건부 렌더링 로직 구현.
2. **Dashboard Component**:
   - 전체 폴더 리스트 데이터를 받아와 '구분(개발/신규/기타)'별로 개수를 카운트하는 가공 로직 작성.
3. **UI/UX 개선**:
   - 대시보드에서 최근 업데이트된 사업 리스트를 클릭하면 해당 폴더가 바로 선택되는 핸들러 연동.
   - 기존의 블루/화이트 테마를 유지하며 통계 수치는 가독성 좋은 큰 폰트와 배지(Badge) 활용.

## 5. 제약 사항
- 초기 진입 시 서버로부터 전체 폴더 데이터를 한 번에 가져오므로, 데이터 로딩 중에는 스켈레톤 UI를 보여줄 것.
- 담당자 정보는 칩(Chip) UI 디자인을 유지할 것.
