# GitDigest 주간보고 스케줄러 팀 단위 고도화 요청서

## 1. 개요
현재 단일 프로세스로 동작하는 주간보고 스케줄러를 **팀(Team) 단위**로 분리하여 관리하고, 사용자의 권한에 따라 차등화된 UI와 기능을 제공하도록 수정한다.

## 2. 주요 요구 사항

### 🛠 팀별 스케줄러 설정 페이지 (관리자 전용)
- **접근 권한**: `ADMIN` 계정만 접근 가능.
- **주요 기능**:
    - 팀별 스케줄러 활성화/비활성화(On/Off) 설정 기능.
    - **스케줄러 주행 시간**: 기존과 동일하게 `.yml` scheduler.weekly-report.cron 설정값을 따르되, On 상태인 팀에 대해서만 동작하도록 로직 수정.

### 수동 실행 UI 및 권한 로직 개선
- **위치**: '주간보고 스케줄러' 페이지 내 [수동 실행] 버튼 옆.
- **권한별 처리**:
    - **ADMIN 계정**: 
        - 팀을 선택할 수 있는 **드롭다운(Dropdown)** 표시.
        - 특정 팀을 선택하여 수동 실행 가능.
    - **일반 계정**: 
        - 드롭다운이 보이지 않음. 
        - 실행 시 본인이 속한 팀이 기본값으로 자동 셋팅되어 실행됨.

### 실행 이력 목록(Table) 수정
- **변경 사항**: 스케줄러 실행 내역 테이블에 **'팀(Team)' 컬럼** 추가.
- **목적**: 각 실행 기록이 어떤 팀을 대상으로 수행되었는지 직관적으로 구분 가능하도록 함.
